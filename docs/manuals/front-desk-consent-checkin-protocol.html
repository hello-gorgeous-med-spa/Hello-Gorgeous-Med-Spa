Consent, Check-In & Client Records

How Square booking, Hello Gorgeous admin, the tablet kiosk, and signed consents fit together — and exactly what front desk does.

Print & keep at front desk. · Hello Gorgeous Med Spa · Updated July 15, 2026

1. The simple mental model

Book / pay→ Client in HG→ Consent packet→ Sign (phone or tablet)→ Locked on client chart
SystemWhat it ownsFront desk truth
Square Online booking calendar, payments, packages “Do they have an appointment / paid?”
Hello Gorgeous Admin Client chart, consent packets, signed PDFs/records, clinical ops “Are consents signed? What’s on file?”
Kiosk / tablet In-spa signing screen only — not a second database Same consents, just signed on the iPad
Email / SMS / link Remote signing of the same packet Same record as tablet — just signed offsite
One vault: Signed consents always live on the Hello Gorgeous client record (/admin/clients → open client → Consents). Tablet, email, SMS, and portal all write to that same place. Square does not replace this vault.

2. How it syncs (what actually happens)

  1. Booking — Client books online (Square /book) or staff books in admin.
  2. Client match — System looks up / creates the HG client by email/phone.
  3. Consent packet — Outstanding forms for that visit/service become a packet tied to the client (and appointment when linked).
  4. Sign — Client signs via:
    • Email / SMS / magic link (at home), or
    • Portal /portal/consents, or
    • Kiosk tablet /kiosk or Calendar → kiosk consent link
  5. Saved — Signature + timestamp attach to the client. Staff see it on the client chart and appointment consent status.
  6. Treat — Provider should not start until required consents show signed (admin may flag unsigned).
Important right now: If an appointment exists only in Square and has not been bridged into Hello Gorgeous, the phone kiosk may say it can’t find them. Fix: open/create the client in HG admin, send consents from /admin/owner/consents or the appointment, or use a generated kiosk link from Calendar when available. Booking money lives in Square; legal signatures live in HG.

3. Front desk protocol — morning

  1. Open calendar: hellogorgeousmedspa.com/admin/calendar
  2. Scan today’s list. Flag:
    • New clients (first visit)
    • New service types (may need extra consents: injectables, laser, weight loss, etc.)
    • Anyone with unsigned / expired consents
  3. Optional: send consent links before they arrive (email/SMS from Consents or appointment) so they sign at home.
  4. Wake the front-desk tablet → bookmark https://www.hellogorgeousmedspa.com/kiosk (Safari, full screen / Guided Access if you use it).

4. When the client walks in

A Preferred — tablet kiosk (in person)

  1. Unlock the tablet → open Kiosk.
  2. Client enters phone number used at booking → Check in.
  3. If consents are outstanding, continue to signing screen (or open the kiosk URL shown).
  4. Hand them the tablet. They read each form and sign with finger/stylus.
  5. When finished, take tablet back. Confirm success message / return to kiosk home.
  6. In admin, open their client → Consents tab → confirm forms show Signed with today’s date.
  7. Mark appointment checked in / notify provider they are ready.

B If they already signed at home

  1. Open client chart → Consents.
  2. Confirm required forms are Signed and not expired.
  3. Check them in. No tablet needed.

C If kiosk can’t find them

  1. Search client in /admin/clients by phone/email/name.
  2. If missing → Create client with correct phone + email.
  3. Go to /admin/owner/consents (or appointment detail):
    • Select the forms needed (at minimum: General + HIPAA; add service-specific)
    • Send via SMS to their phone or Copy link and open that link on the tablet
  4. They sign → refresh client Consents tab → verify Signed.

D Remote / no-show prevention (send ahead)

  1. /admin/owner/consents → choose form(s) → Email or SMS.
  2. Or from appointment: send consent / resend outstanding packet.
  3. Tell client: “Please complete the secure link before you arrive.”

5. What “keep it safe” means for front desk

  • Do use HG consent links / kiosk only — never photo paper consents into random texts as the “official” file.
  • Do verify Signed on the client chart before rooming for treatment.
  • Do keep the tablet on the HG kiosk / consent URL only during check-in (no personal browsing with client PHI on screen).
  • Don’t email full medical histories to your personal inbox.
  • Don’t assume Square “Contracts” = your medical consent vault (optional booking forms ≠ HG signed consents).
  • Lock the tablet when not in use (passcode / Guided Access).

8. RX / RE GEN note (don’t mix with injectables check-in)

Weight-loss / peptide / RE GEN intakes (/glp1-intake, /peptide-request, /rx/checkout/intake) are a separate clinical track from same-day injectable consents. Front desk: if they’re here for aesthetics, use consents above. If they paid for RE GEN online, make sure post-payment intake is done before staff promises ship dates — ask admin/RX desk if unsure.

9. End-of-day (30 seconds)

  • Any clients who were treated with unsigned required consents? Fix today (send + get signed), document with owner if treatment already happened.
  • Plug tablet in; leave kiosk bookmark ready for tomorrow.
  • Log any “kiosk couldn’t find phone” cases so ops can fix Square ↔ HG appointment sync gaps.
One sentence to remember: Square books the visit · Hello Gorgeous holds the signed legal forms · The tablet is just how they sign in the lobby.
